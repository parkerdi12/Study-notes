git init - создание пустого репозитория git checkout -b - создание новой ветки git checkout master $ git checkout develop $ git checkout my_feature - переключение веток git branch - вывести список веток git branch -d - удалить лок. ветку git clone ссылка на репозиторий - клонируем репозиторий к себе на локальную машину cd название папки - переход в папку "cd" - choose directory git add название файла - обновить файл, добавить в проект (проиндексировать) git commit -m "first commit" - внесение изменений перед push git push - отправка коммита на сервер git pull - забрать изменения для мастера git merge - интегрировать коммиты в основную ветку Команда git commit делает для проекта снимок текущего состояния изменений, добавленных в раздел проиндексированных файлов. Такие подтвержденные снимки состояния можно рассматривать как «безопасные» версии проекта — Git не будет их менять, пока вы явным образом не попросите об этом. Перед выполнением команды git commit необходимо использовать команду git add, чтобы добавить в проект («проиндексировать») изменения, которые будут сохранены в коммите. Эти две команды, git commit и git add, используются чаще всего. Команда git push используется для выгрузки содержимого локального репозитория в удаленный репозиторий. Она позволяет передать коммиты из локального репозитория в удаленный.
